# Publication Review Wrapper Guidance

Keep publication command wrappers thin. State the command role, include the workflow link, keep context concise, and defer stage ordering, retries, manuscript-root preflight, and schema visibility to the linked workflow. Do not restate the full publication pipeline in the wrapper.
When wrappers need publication bootstrap or manuscript-root gating, point to `publication-bootstrap-preflight.md` and the canonical manuscript-root, round-artifact, and response-artifact references instead of re-embedding those rules inline.
When wrappers need response drafting or paired response completion, point to `publication-response-writer-handoff.md` and the canonical response-artifact reference instead of repeating the handoff contract inline.
