Keep the beginner boundaries stable:

- `start` is the chooser/router for the safest first path in the current folder.
- `tour` is the read-only explainer; it does not run setup, resume, or configuration commands.
- `help` is the reference/index surface, not a setup wizard.
- `settings` belongs after first successful startup or when the user explicitly asks to change autonomy, permissions, workflow defaults, model-cost posture, or tier overrides.

A common first pass is `help -> start -> tour -> new-project / map-research -> resume-work`, but the folder state still decides the actual path.

GPD is a scalpel, not an autopilot. Trust agent execution, but verify and redirect. Use Supervised for frequent checkpoints; move to Balanced once you trust the boundary.
