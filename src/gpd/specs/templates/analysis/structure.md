# Research Project Structure

**Analysis Date:** 2025-06-15

## Directory Layout
```

pn-waveforms/
├── derivations/ # Symbolic derivations (Mathematica, SymPy)
│ ├── energy/ # Orbital energy and binding energy
│ ├── flux/ # Energy and angular momentum flux
│ └── waveform/ # Waveform multipole moments
├── src/ # Computational code (Python)
│ ├── dynamics/ # Orbital evolution
│ ├── waveform/ # Waveform generation
│ ├── analysis/ # Post-processing and parameter estimation
│ └── symbolic/ # Python-based symbolic utilities
├── tests/ # Test suite
│ ├── unit/ # Unit tests for individual functions
│ └── integration/ # End-to-end validation tests
├── data/ # Input data (not generated)
│ ├── noise_curves/ # Detector PSDs
│ └── nr_waveforms/ # Numerical relativity comparison data
├── configs/ # Run configuration files
├── notebooks/ # Exploratory Jupyter notebooks
├── figures/ # Publication figures
├── output/ # Generated results (gitignored)
├── paper/ # Paper draft (LaTeX)
├── pyproject.toml # Project configuration
└── README.md # Project overview

```

## Directory Purposes

**derivations/**
- Purpose: Symbolic derivations of all analytical results
- Contains: Mathematica .nb notebooks, SymPy .py scripts
- Key files: `flux/pn_energy_flux.nb` (main flux derivation), `energy/pn_binding_energy.nb` (orbital energy)
- Subdirectories: Organized by physical quantity (energy, flux, waveform)

**src/**
- Purpose: All computational Python code
- Contains: Python modules organized by function
- Key files: `dynamics/orbital_evolution.py` (ODE integrator), `waveform/generator.py` (main waveform code)
- Subdirectories: dynamics/ (orbit), waveform/ (GW signal), analysis/ (post-processing), symbolic/ (CAS utilities)

**tests/**
- Purpose: Automated verification of all computational code
- Contains: pytest test files with @pytest.mark.unit and @pytest.mark.integration markers
- Key files: `unit/test_pn_coefficients.py` (coefficient verification), `integration/test_waveform_accuracy.py` (end-to-end)
- Subdirectories: unit/ (fast, isolated), integration/ (slower, cross-module)

**data/**
- Purpose: External input data (not generated by this project)
- Contains: Detector noise curves, NR waveforms for comparison
- Key files: `noise_curves/aLIGO_O3_psd.dat`, `nr_waveforms/SXS_BBH_0305.h5`
- Subdirectories: By data source type

**notebooks/**
- Purpose: Exploratory analysis, prototyping, visualizations
- Contains: Jupyter notebooks (.ipynb) for interactive work
- Key files: `resummation_study.ipynb`, `parameter_space_survey.ipynb`
- Note: Results from notebooks should be promoted to src/ once validated

**paper/**
- Purpose: LaTeX source for publication
- Contains: .tex files, .bib bibliography, figure references
- Key files: `main.tex`, `references.bib`

## Key File Locations

**Derivations:**
- `derivations/flux/pn_energy_flux.nb`: Energy flux to 3.5PN order
- `derivations/energy/pn_binding_energy.nb`: Binding energy to 4PN order
- `derivations/waveform/multipole_moments.nb`: Radiative multipole moments
- `derivations/flux/tail_integrals.nb`: Hereditary tail contributions

**Computational Code:**
- `src/dynamics/orbital_evolution.py`: ODE integration for orbital phase
- `src/dynamics/pn_equations.py`: PN equations of motion and energy balance
- `src/waveform/generator.py`: Main waveform generation pipeline
- `src/waveform/frequency_domain.py`: Time-to-frequency domain conversion
- `src/analysis/fisher.py`: Fisher matrix for parameter estimation forecasts

**Data:**
- `data/noise_curves/aLIGO_O3_psd.dat`: Advanced LIGO O3 noise curve
- `data/noise_curves/ET_D_psd.dat`: Einstein Telescope design sensitivity
- `data/nr_waveforms/`: SXS catalog waveforms (HDF5)

**Configuration:**
- `pyproject.toml`: Project dependencies and metadata
- `configs/run_config.yaml`: Default parameters for production runs
- `configs/parameter_space.json`: Grid of parameters for surveys

**Results and Figures:**
- `figures/`: Publication PDF figures (committed)
- `output/`: Generated numerical results (gitignored)
- `output/waveform_banks/`: Template banks (gitignored)

## Naming Conventions

**Files:**
- snake_case.py: Python source modules (orbital_evolution.py)
- snake_case.nb: Mathematica notebooks (pn_energy_flux.nb)
- test_*.py: Test files (test_pn_coefficients.py)
- fig_*.pdf: Publication figures (fig_phase_comparison.pdf)
- UPPERCASE.md: Important project files (README.md, CONVENTIONS.md)

**Directories:**
- snake_case: All directories
- Plural for collections: derivations/, figures/, tests/, notebooks/
- Singular for functional units: src/dynamics/, src/waveform/

**Special Patterns:**
- __init__.py: Python package marker
- conftest.py: pytest fixture definitions
- *.ipynb: Jupyter notebooks (exploratory work)

## Where to Add New Work

**New derivation:**
- Notebook/notes: `derivations/[topic]/[description].nb`
- Summary: Add key results to THEORETICAL_FRAMEWORK.md
- Verification: Add numerical check in `tests/unit/test_[topic]_coefficients.py`

**New computational module:**
- Implementation: `src/[subsystem]/[module_name].py`
- Tests: `tests/unit/test_[module_name].py`
- Configuration: Add parameters to `configs/run_config.yaml` if needed

**New dataset:**
- Raw data: `data/[source_type]/[filename]`
- Processed data: `output/processed/[filename]` (gitignored)
- Documentation: Add entry to CONNECTIONS.md

**New figure:**
- Plotting script: `notebooks/plot_[description].ipynb` or `src/analysis/plot_[description].py`
- Output: `figures/fig_[description].pdf`
- Data source: Reference in notebook which output/ files are used

## Special Directories

**output/**
- Purpose: All generated numerical results
- Source: Produced by computational code in src/
- Committed: No (in .gitignore) - regenerate with `make run`

**data/nr_waveforms/**
- Purpose: External NR waveform data for calibration/validation
- Source: Downloaded from SXS catalog (sxs.caltech.edu)
- Committed: No (in .gitignore) - download with `make download-data`

<guidelines>
**What belongs in STRUCTURE.md:**
- Directory layout (ASCII box-drawing tree for structure visualization)
- Purpose of each directory
- Key file locations (derivations, code, data, configs)
- Naming conventions
- Where to add new work (by type)
- Special/generated directories

**What does NOT belong here:**

- Theoretical framework (that's THEORETICAL_FRAMEWORK.md)
- Methods and tools (that's METHODS.md)
- Notation conventions (that's CONVENTIONS.md)
- Every single file (focus on directories and key files)

**When filling this template:**

- Use `tree -L 2` or similar to visualize structure
- Identify top-level directories and their purposes
- Note naming patterns by observing existing files
- Locate derivations, code, data, and output areas
- Keep directory tree concise (max 2-3 levels)

**Useful for research planning when:**

- Adding new derivations or code (where should files go?)
- Understanding project organization
- Finding where specific calculations live
- Following existing conventions
  </guidelines>
