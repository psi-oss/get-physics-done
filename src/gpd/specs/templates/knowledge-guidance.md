# Knowledge Guidance

Document authors can always refer to this note when they need clarity about which
behaviors remain deferred from the current `knowledge` workflow.

## Deferred Behaviors

The following capabilities are intentionally excluded from the active knowledge
authoring surface until a dedicated migration or feature effort proves them out:

- migration/backfill for older or provisional docs
- alias repair or filename/ID normalization for legacy docs
- beginner onboarding exposure
- `knowledge_deps` and `related_artifacts` frontmatter support
- implicit discovery outside the canonical `knowledge` layout contract
- automatic promotion of a draft to stable without review

When any of these behaviors ships, this note should be updated and the template
can adopt the changes by pointing elsewhere instead of carrying the list inline.
